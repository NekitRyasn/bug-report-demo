Демонстрационная форма баг-репорта (пример для AQA Java)
Заглушка — загрузка файлов не реализована
Пояснение для AQA Java:
Структура отчёта соответствует стандартному баг-репорту. Поля Steps, Expected и Actual используются для построения тест-кейсов и автоматизированных проверок (assertions). Форма подходит для UI-автотестов на Java с применением Selenium / Selenide и Page Object.
Страница является демонстрационной. Серверная логика, сохранение данных и загрузка вложений отсутствуют.
